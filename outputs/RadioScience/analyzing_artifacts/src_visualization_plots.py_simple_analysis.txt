## Logic Analysis: `src/visualization/plots.py`

### 1. Purpose of This Module

`src/visualization/plots.py` implements the `Plotter` class defined in the design. Its role is to generate publication-style diagnostic and reproduction figures **after** all scientific computation has already been completed by upstream modules.

This module must **not** perform core geolocation, QC, propagation, reprocessing, or filtering logic. It should only perform:

- figure formatting,
- display-oriented reshaping of already-computed arrays/DataFrames,
- plotting of precomputed curves and binned products,
- writing image files to the requested output path.

The module supports visual reproduction of paper-style outputs:

1. Synthetic propagation diagnostics:
   - spectra,
   - single-screen and two-screen `V(L)` curves,
   - orientation-error behavior,
   - thermal-noise `V(L)` curves,
   - multivalued geometry examples.

2. Real-data diagnostics:
   - `V(L)` curves,
   - `D(L)` curves,
   - `cos(alpha)` curves.

3. Final real-data products:
   - monthly COSMIC-2 geolocation density maps,
   - L1/L2 zonal-difference histogram,
   - F10.7 solar flux time series.

The public API must follow the design exactly:

```text
class Plotter {
    +plot_v_curve(curve: BpCurve, output_path: Path) None
    +plot_d_curve(d_curve: DCurve, output_path: Path) None
    +plot_monthly_maps(binned: DataFrame, output_dir: Path) None
    +plot_l1_l2_histogram(pairs: DataFrame, output_path: Path) None
    +plot_f107(f107: DataFrame, output_path: Path) None
    +plot_synthetic_outputs(results: dict, output_dir: Path) None
}
```

No additional public methods should be introduced.

---

## 2. Configuration Alignment

Although the `Plotter` interface in the design does not take `AppConfig` directly, all visual products must reflect values specified in `config.yaml` and generated by upstream modules using that configuration.

Relevant configuration values are:

### 2.1 Monthly map settings

From:

```yaml
real_data_experiments:
  monthly_l1_maps:
    local_time_hours:
      - 18
      - 24
    bin_size_deg:
      latitude: 3
      longitude: 3
```

and:

```yaml
processing:
  local_time_filter:
    enabled_for_monthly_maps: true
    start_hour: 18
    end_hour: 24
    reference: "geolocation_longitude"
```

Implications for `plot_monthly_maps`:

- The input `binned` DataFrame should already contain accepted, post-sunset geolocations.
- The local-time filter should already have been applied upstream by `RealDataExperiments.make_monthly_bins`.
- The bin size should already be 3° × 3°.
- The plot title/labels should state that maps represent post-sunset 18–24 LT geolocations if metadata or column context indicates this product.

### 2.2 L1/L2 histogram settings

From:

```yaml
real_data_experiments:
  l1_l2_comparison:
    zonal_difference_histogram_bin_deg: 0.5
```

Implication:

- `plot_l1_l2_histogram` should use 0.5° histogram bins for L1-L2 zonal difference, matching the paper’s Figure 23-style comparison.

### 2.3 Back-propagation diagnostic settings

From:

```yaml
back_propagation:
  distance_grid:
    min_km: 100
    max_km: 6000
    step_km: 100

  magnetic_field_candidate_grid:
    min_km: 100
    max_km: 6000
    step_km: 100

  v_curve:
    definition: "<A^2> / <A>^2 - 1"
    smoothing:
      method: "second_order_sliding_polynomial_regression"
      window_km: 1000
      polynomial_order: 2
```

Implications:

- `plot_v_curve` should label the x-axis as distance from receiver toward transmitter, in km.
- It should plot both raw and smoothed `V(L)` if both are available in `BpCurve`.
- The smoothed curve corresponds to the configured 1000 km, second-order sliding polynomial regression.
- It should mark the global local minimum `L0`.
- It should optionally visualize `L1`, `L2`, `V1`, `V2`, and `Q` using fields already stored in `BpCurve`.

### 2.4 Synthetic experiment settings

From:

```yaml
synthetic_experiments:
  phase_screen:
    spectral_index_p: 1.5
    outer_scale_km: 10

  observed_average_values:
    sigma_phi_rad: 1.65
    s4: 0.32
    sigma_phi_over_s4_rad: 5.25
    distance_from_phase_screen_to_receiver_km: 1900
    estimated_outer_scale_km: 9.7

  modeled_cases:
    thermal_noise:
      snr_1hz_cases_vv:
        - 200
        - 400
        - 600
      sigma_phi_cases_rad:
        - 1.65
        - 0.25
```

Implications:

- Synthetic plots should annotate or title curves using the values actually contained in `results`.
- If labels are needed and values are present, use the paper/config values:
  - `p = 1.5`,
  - `l0 = 10 km`,
  - `sigma_phi = 1.65 rad`,
  - `sigma_phi = 0.25 rad`,
  - `L = 1900 km` or `2000 km` where applicable,
  - SNR values 200, 400, 600 V/V.

## 4. General Plotting Principles

### 4.1 No scientific recomputation

This module should not:

- recompute `V(L)`,
- smooth `V(L)`,
- find local minima,
- compute `D(L)`,
- perform zero-crossing detection,
- apply QC thresholds,
- filter accepted/rejected geolocations,
- compute geolocations,
- compute scintillation indices,
- run propagation,
- re-bin raw geolocation data if the method receives already-binned data.

Allowed display operations:

- converting arrays to NumPy arrays for plotting,
- reshaping binned monthly-map data into 2D grids,
- computing histogram bin edges for display in `plot_l1_l2_histogram`,
- selecting DataFrame columns,
- grouping already-binned map products by month/year for subplots,
- setting axis limits from data range.

### 4.2 Units

The module must preserve units used by upstream structures:

| Quantity | Unit |
|---|---|
| BP distance | km |
| Magnetic-field candidate distance `L_mf` | km |
| `D(L_mf)` | km |
| Latitude / longitude | degrees |
| Zonal L1-L2 difference | degrees |
| F10.7 | solar flux units, typically sfu |
| `V(L)` | dimensionless |
| `cos(alpha)` | dimensionless |

### 4.3 Output behavior

Each plotting method should:

1. Ensure the output directory exists.
2. Create a figure with readable size.
3. Save the figure to the requested path.
4. Close the figure after saving to avoid memory leaks during batch processing.

This is especially important for monthly maps and full-year processing, where many figures may be generated.

## 5.1 `plot_v_curve(curve: BpCurve, output_path: Path) -> None`

### Purpose

Plot the normalized back-propagated amplitude variance:

\[
V(L) = \frac{\langle A^2\rangle}{\langle A\rangle^2} - 1
\]

as a function of back-propagation distance \(L\).

This corresponds to diagnostics like Figures 16 and 17 in the paper.

### Input contract

`curve` is a `BpCurve` with fields:

```text
curve.l_mf_km
curve.distances_km
curve.v_raw
curve.v_smooth
curve.l0_km
curve.v0
curve.l1_km
curve.l2_km
curve.v1
curve.v2
curve.q
curve.has_valid_minimum
```

### Plot content

The figure should include:

1. Raw `V(L)` curve:
   - x-axis: `curve.distances_km`
   - y-axis: `curve.v_raw`
   - line style: thin gray or black.

2. Smoothed `V(L)` curve:
   - x-axis: `curve.distances_km`
   - y-axis: `curve.v_smooth`
   - line style: thicker red or blue.
   - This smoothed curve corresponds to the configured:
     - 1000 km smoothing window,
     - second-order sliding polynomial regression.

3. Minimum marker:
   - vertical line at `curve.l0_km`,
   - point marker at `(curve.l0_km, curve.v0)`.

4. Optional `L1`, `L2` markers:
   - if `curve.l1_km` and `curve.l2_km` are valid:
     - mark `curve.l0_km - curve.l1_km`,
     - mark `curve.l0_km + curve.l2_km`,
     - optionally draw vertical dashed lines.

5. Title or annotation:
   - include `L_mf`,
   - include `L0`,
   - include `Q`,
   - include whether the minimum is valid.

Example title content:

```text
V(L), L_mf = 1900 km, L0 = 2000 km, Q = 1.35
```

### Axis labels

```text
x-axis: Back-propagation distance L from receiver (km)
y-axis: Normalized amplitude variance V(L)
```

### Fidelity to paper

The plot should visually support the paper’s logic:

- `V(L)` decreases toward the phase screen,
- reaches a minimum near `L0`,
- increases beyond the screen,
- smoothed `V(L)` is used for robust `Q` estimation.

### Edge cases

If `curve.v_smooth` is missing, empty, or all NaN:

- plot only `v_raw`,
- label clearly that smoothed curve is unavailable.

If `curve.has_valid_minimum` is false:

- still plot the curve,
- annotate “invalid minimum” or similar,
- do not imply accepted geolocation.

## 5.2 `plot_d_curve(d_curve: DCurve, output_path: Path) -> None`

### Purpose

Plot the geolocation consistency function:

\[
D(L_{\mathrm{mf}}) = L_0 - L_{\mathrm{mf}}
\]

and optionally the corresponding `cos(alpha)` curve.

This corresponds to diagnostics like Figures 18 and 19 in the paper.

### Input contract

`d_curve` is a `DCurve` with fields:

```text
d_curve.l_mf_km
d_curve.d_km
d_curve.l0_km
d_curve.q
d_curve.cos_alpha
d_curve.zero_crossings_km
d_curve.is_multivalued
```

### Plot content

A useful diagnostic layout is two stacked panels.

#### Panel 1: `D(L_mf)`

Plot:

```text
x = d_curve.l_mf_km
y = d_curve.d_km
```

Include:

- horizontal zero line at `D = 0`,
- vertical lines at each value in `d_curve.zero_crossings_km`,
- markers for valid zero crossings,
- annotation if `d_curve.is_multivalued` is true.

Axis labels:

```text
x-axis: Magnetic-field candidate distance L_mf (km)
y-axis: D = L0 - L_mf (km)
```

#### Panel 2: `cos(alpha)`

Plot:

```text
x = d_curve.l_mf_km
y = d_curve.cos_alpha
```

Include:

- horizontal reference line at configured QC threshold:

\[
\cos\alpha = 0.1
\]

from:

```yaml
back_propagation:
  quality_control:
    cos_alpha_threshold: 0.1
```

Axis labels:

```text
x-axis: Magnetic-field candidate distance L_mf (km)
y-axis: cos(alpha)
```

### Fidelity to paper

This diagnostic should make clear:

- accepted geolocation is where `D(L_mf)` crosses zero,
- multiple zero crossings imply multivalued geolocation,
- multivalued geolocations are discarded upstream according to config:

```yaml
back_propagation:
  quality_control:
    require_single_valued_geolocation: true
    discard_multivalued_geolocations: true
```

### Edge cases

If there are no zero crossings:

- plot the curve normally,
- annotate “no zero crossing”.

If multiple zero crossings exist:

- mark all crossings,
- annotate “multivalued”.

If `cos_alpha` is missing:

- plot only the `D` panel.

## 5.3 `plot_monthly_maps(binned: DataFrame, output_dir: Path) -> None`

### Purpose

Plot monthly latitude/longitude density maps of accepted COSMIC-2 geolocations.

This corresponds to Figure 21 in the paper.

### Input contract

The input `binned` DataFrame should already be generated by:

```text
RealDataExperiments.make_monthly_bins(results)
```

The input should already reflect:

- accepted geolocations only,
- post-sunset local time 18–24 hr,
- geolocation-longitude local-time reference,
- 3° × 3° latitude/longitude bins,
- signal selection, usually L1,
- target years 2021 and 2023.

Expected columns should include enough information to identify:

- year,
- month,
- latitude bin or latitude bin center,
- longitude bin or longitude bin center,
- count.

The exact column names are not specified by the design, so implementation should either:

1. standardize expected column names in `RealDataExperiments.make_monthly_bins`, or
2. defensively support a small set of likely names.

Recommended standardized columns:

```text
year
month
lat_bin_center_deg
lon_bin_center_deg
count
```

or:

```text
year
month
latitude_bin
longitude_bin
count
```

### Plot structure

Recommended reproduction layout:

- one figure per year,
- 12 monthly panels per year,
- arranged as 3 rows × 4 columns or 4 rows × 3 columns.

For each panel:

- use a Plate Carree projection,
- plot counts in 3° × 3° bins,
- include coastlines,
- include longitude/latitude gridlines,
- title each panel with month name and year.

### Map projection

Use:

```text
cartopy.crs.PlateCarree()
```

This is appropriate because the paper uses geographic latitude/longitude maps.

### Color scale

The paper uses color scale as number of geolocations per 3° × 3° bin.

Implementation should:

- use counts directly,
- use a shared color scale within each year figure to allow month-to-month comparison,
- optionally use a shared color scale across 2021 and 2023 if both years are plotted together, but this method receives one DataFrame and no config, so safest behavior is shared scale within the plotted subset.

### Axis extent

The paper focuses on equatorial and low/middle latitudes because COSMIC-2 has low inclination. However, `config.yaml` does not specify latitude limits.

Therefore:

- do not hard-code an equatorial latitude range unless the binned data itself is limited,
- use global longitude range,
- use the latitude range present in `binned`, or a global map if needed.

### Labels and title

Title should indicate:

```text
COSMIC-2 L1 geolocations, post-sunset 18–24 LT, 3° × 3° bins
```

when plotting the main monthly product.

These values align with:

```yaml
real_data_experiments:
  monthly_l1_maps:
    signal: "L1"
    local_time_hours:
      - 18
      - 24
    bin_size_deg:
      latitude: 3
      longitude: 3
```

### Fidelity targets

The resulting maps should allow comparison to the paper’s reported patterns:

- 2021 total accepted L1 geolocations: about 332,000.
- 2023 total accepted L1 geolocations: about 1,108,000.
- Seasonal morphology:
  - December–January: American/Atlantic sector,
  - May–July: African and Pacific sectors,
  - September–October: broad longitude coverage.

These totals and morphology are not computed in this plotting method but should be visible if upstream processing is faithful.

### Edge cases

If a month has no rows:

- create an empty map panel,
- title it with the month,
- optionally annotate “no data”.

If only one month/year is present:

- plot only the available subset.

## 5.4 `plot_l1_l2_histogram(pairs: DataFrame, output_path: Path) -> None`

### Purpose

Plot the histogram of L1-L2 zonal geolocation differences.

This corresponds to Figure 23b in the paper.

### Input contract

The input `pairs` DataFrame should already be produced by:

```text
RealDataExperiments.pair_l1_l2(l1_results, l2_results)
```

It should contain paired geolocations from the same:

- event ID,
- LEO satellite,
- GNSS transmitter,
- 10-s interval.

The DataFrame should include a precomputed zonal difference in degrees.

Recommended column name:

```text
zonal_difference_deg
```

Alternative acceptable names could include:

```text
delta_lon_deg
l1_minus_l2_lon_deg
```

The histogram represents:

\[
\Delta \lambda = \lambda_{L1} - \lambda_{L2}
\]

in degrees.

### Histogram bins

Use bin width:

\[
0.5^\circ
\]

from:

```yaml
real_data_experiments:
  l1_l2_comparison:
    zonal_difference_histogram_bin_deg: 0.5
```

Because the `Plotter` constructor does not receive `AppConfig`, this value should be treated as the paper/config-defined visual convention for this reproduction figure.

### Plot content

- Histogram of L1-L2 zonal difference.
- Vertical line at zero.
- Optional annotation of:
  - number of paired cases,
  - mean difference,
  - median difference,
  - standard deviation.

Mean/median/std are display summaries only. If strict “no computation beyond plotting” is desired, these should either:
- be omitted, or
- only displayed if already present in `pairs.attrs` or provided columns.

### Axis labels

```text
x-axis: L1 - L2 zonal geolocation difference (deg)
y-axis: Number of paired 10-s geolocations
```

### Fidelity targets

The paper reports for the comparison period configured as:

```yaml
signals:
  l1_l2_comparison:
    start_date: "2021-02-19"
    end_date: "2021-03-01"
```

Expected counts:

```yaml
real_data_experiments:
  l1_l2_comparison:
    expected_counts:
      l1_geolocations: 13000
      l2_geolocations: 11000
      common_l1_l2_cases: 8000
```

The histogram should be centered near zero if the reproduction is successful.

### Edge cases

If the zonal-difference column is missing:

- raise a clear error,
- do not silently compute a different quantity unless the implementation explicitly supports known alternative columns.

If the DataFrame is empty:

- create an empty histogram figure with an annotation,
- save it normally.

## 5.5 `plot_f107(f107: DataFrame, output_path: Path) -> None`

### Purpose

Plot daily F10.7 solar flux values.

This corresponds to Figure 22 in the paper.

### Input contract

The input `f107` DataFrame should come from:

```text
F107Loader.download(start, end)
```

or:

```text
F107Loader.load_cached(path)
```

Expected columns:

```text
date
f107
```

or compatible names.

The data source is configured as:

```yaml
data:
  f107:
    source: "https://omniweb.gsfc.nasa.gov/form/dx1.html"
```

### Plot content

- line plot of daily F10.7 versus date,
- date on x-axis,
- F10.7 on y-axis,
- optional highlighting or vertical spans for 2021 and 2023.

Highlighting 2021 and 2023 is useful because the paper compares low and high solar activity between these years.

From config:

```yaml
data:
  cosmic2:
    years:
      - 2021
      - 2023
```

### Axis labels

```text
x-axis: Date
y-axis: F10.7 solar flux
```

### Fidelity target

The visual trend should show that F10.7 increased substantially from 2021 to 2023, consistent with the paper’s explanation for higher scintillation/geolocation counts in 2023.

### Edge cases

If the F10.7 DataFrame is empty:

- save an empty plot with an annotation.

If the date column is not parsed as datetime:

- convert it for plotting only.

## 5.6 `plot_synthetic_outputs(results: dict, output_dir: Path) -> None`

### Purpose

Generate figures from synthetic experiment outputs.

This method is called after:

```text
SyntheticExperiments.run_all()
```

The method should plot whatever synthetic result dictionaries are available without recomputing the experiments.

### Expected synthetic experiment keys

The exact dictionary schema is not fixed in the design. However, it should correspond to methods in `SyntheticExperiments`:

```text
single_phase_screen
outer_scale_estimation
orientation_error
two_screens_aligned
two_screens_misaligned
thermal_noise
multivalued_geometry
```

The plotting method should check for these keys and create figures for each available result.

---

### 5.6.1 Single phase screen plot

Expected content:

- `V(L)` curve,
- true phase-screen distance marker,
- recovered minimum marker.

Purpose:

- verify that the minimum of `V(L)` occurs at the screen location.

Relevant config values:

```yaml
synthetic_experiments:
  observed_average_values:
    distance_from_phase_screen_to_receiver_km: 1900
```

and:

```yaml
synthetic_experiments:
  modeled_cases:
    thermal_noise:
      phase_screen_distance_km: 2000
```

depending on which experiment generated the result.

---

### 5.6.2 Outer-scale estimation plot

Expected content:

- curves of `sigma_phi / S4` versus outer scale `l0`,
- separate curves for multiple screen-receiver distances,
- horizontal reference line at:

\[
\sigma_\phi/S_4 = 5.25 \text{ rad}
\]

from:

```yaml
synthetic_experiments:
  observed_average_values:
    sigma_phi_over_s4_rad: 5.25
```

- vertical reference line near:

\[
l_0 \approx 9.7 \text{ km}
\]

from:

```yaml
synthetic_experiments:
  observed_average_values:
    estimated_outer_scale_km: 9.7
```

Purpose:

- reproduce the logic of Figure 3b.

---

### 5.6.3 Orientation-error plot

Expected content:

- geolocation error versus true distance,
- separate curves for scan angles:
  - 15°,
  - 45°,
  - 75°.

From config:

```yaml
synthetic_experiments:
  modeled_cases:
    orientation_error:
      delta_alpha_deg: 0.6
      table_cases:
        - alpha_deg: 15
        - alpha_deg: 45
        - alpha_deg: 75
```

The plotted values should already be computed upstream using:

\[
\Delta L = 2L\tan(\alpha)\Delta\alpha
\]

This method should only plot the resulting arrays/table.

Purpose:

- reproduce the logic of Table 1 and Figure 5.

---

### 5.6.4 Two aligned phase screens plot

Expected content:

- `V(L)` for screen 1 only,
- `V(L)` for screen 2 only,
- `V(L)` for both screens,
- vertical lines at true screen locations.

For unequal-strength case:

- indicate stronger/weaker screens,
- ratio should reflect config:

```yaml
synthetic_experiments:
  modeled_cases:
    two_phase_screens_same_orientation:
      sigma_phi_ratio_stronger_to_weaker: 1.5
```

Purpose:

- show that aligned phase screens are not separately resolved,
- show stronger screen dominates localization.

---

### 5.6.5 Two misaligned phase screens plot

Expected content:

- `V(L)` curves for the two scan-angle configurations:
  - `[30, 60]`,
  - `[60, 30]`.

From config:

```yaml
synthetic_experiments:
  modeled_cases:
    two_phase_screens_different_orientation:
      alpha_cases_deg:
        - [30, 60]
        - [60, 30]
```

Purpose:

- show that the screen with smaller scan angle is localizable.

---

### 5.6.6 Thermal-noise plot

Expected content:

- `V(L)` curves for:
  - `sigma_phi = 1.65 rad`,
  - `sigma_phi = 0.25 rad`,
  - SNR = 200, 400, 600 V/V.

From config:

```yaml
synthetic_experiments:
  modeled_cases:
    thermal_noise:
      sigma_phi_cases_rad:
        - 1.65
        - 0.25
      snr_1hz_cases_vv:
        - 200
        - 400
        - 600
      phase_screen_distance_km: 2000
```

Purpose:

- reproduce Figure 11-style behavior:
  - noise raises the `V(L)` baseline,
  - strong scintillation remains detectable,
  - threshold-level scintillation can become marginal at low SNR.

---

### 5.6.7 Multivalued geometry plot

Expected content:

- degenerate or modeled `cos(alpha)(L)` curves,
- `V(L)` curves associated with multiple apparent geolocations,
- markers at zero-crossing or candidate geolocation distances.

Purpose:

- support the paper’s conclusion that multivalued geolocations arise from scan-angle geometry, not from multiple physical irregularity regions.

## 7. Visual Style Recommendations

To make figures consistent and reproducible:

### 7.1 General

- Use clear axis labels with units.
- Use legends only where needed.
- Use gridlines for line plots.
- Use tight layout or constrained layout.
- Save figures with sufficient resolution, e.g. 150–300 dpi.

### 7.2 Maps

- Use `PlateCarree`.
- Add coastlines.
- Add borders lightly if desired.
- Use a perceptually reasonable colormap such as:
  - `viridis`,
  - `plasma`,
  - `magma`,
  - or another sequential colormap.
- Counts should be plotted with a sequential scale.
- Avoid diverging colormaps for count data.

### 7.3 Diagnostic curves

- Raw `V(L)`: black/gray thin line.
- Smoothed `V(L)`: red or blue thicker line.
- Minimum or crossing: dashed vertical line.
- QC threshold lines:
  - `D = 0`,
  - `cos(alpha) = 0.1`,
  - histogram zero line.

## 9. Validation Checks for This Module

The plotting module can be validated without real COSMIC-2 data by using mock objects/DataFrames.

### 9.1 `plot_v_curve`

Test with a synthetic `BpCurve` where:

- distances are 100–6000 km,
- `V(L)` has a known minimum,
- `l0_km` is inside the range,
- `q` is finite.

Expected:

- output file exists,
- minimum line appears,
- no crash when `v_smooth` is missing.

### 9.2 `plot_d_curve`

Test with a `DCurve` containing:

- one zero crossing,
- multiple zero crossings,
- no zero crossings.

Expected:

- crossing markers appear correctly,
- multivalued annotation appears when `is_multivalued = true`,
- `cos(alpha) = 0.1` threshold line appears when `cos_alpha` exists.

### 9.3 `plot_monthly_maps`

Test with a small binned DataFrame containing:

- one year,
- one or two months,
- several lat/lon bins.

Expected:

- one or more map images are written,
- count colorbar appears,
- empty months do not crash.

Test with a DataFrame containing:

```text
zonal_difference_deg
```

Expected:

- histogram uses 0.5° bins,

- empty DataFrame still produces a valid figure.

### 9.5 `plot_f107`

Test with a date range and F10.7 values.

Expected:

- time series figure is written,

### 9.6 `plot_synthetic_outputs`

Test with a partial `results` dictionary.

Expected:

- method plots only available keys,

- no synthetic computation is performed inside plotting.

## 10. Key Implementation Constraints

1. **Do not change the public `Plotter` interface** from the design.

2. **Do not perform scientific computation** in this module beyond display-level operations.

3. **Use config-derived values consistently**, especially:
   - 3° × 3° map bins,
   - 18–24 LT post-sunset product label,
   - 0.5° L1/L2 histogram bins,
   - `cos(alpha) > 0.1` threshold reference line,
   - BP distance units in km,
   - smoothed `V(L)` interpretation as 1000 km second-order polynomial smoothing.

4. **Plot accepted products as produced upstream**. The plotter should not decide acceptance or rejection.

5. **Close figures after saving** to support full-year batch processing.

6. **Keep Cartopy usage isolated to map plotting** so line-plot diagnostics remain lightweight.

---

## 11. Summary

`src/visualization/plots.py` is a downstream visualization layer. It should faithfully display the outputs of the synthetic and real COSMIC-2 back-propagation pipeline without modifying the scientific results.

The most important reproduction-facing responsibilities are:

- showing `V(L)` minima and `Q` diagnostics,
- showing `D(L_mf)` zero crossings and multivalued cases,
- mapping accepted post-sunset L1 geolocation counts in 3° × 3° monthly bins,
- plotting the L1-L2 zonal-difference histogram with 0.5° bins,
- plotting F10.7 solar activity for contextual comparison,
- visualizing synthetic validation experiments corresponding to the paper’s methodological figures.
