## Logic Analysis: `src/propagation/fft_propagator.py`

### 1. Purpose of This Module

`src/propagation/fft_propagator.py` implements the FFT-based plane-wave propagation engine used by both:

1. **Synthetic validation experiments**, reproducing Section 2 of the paper.
2. **Real COSMIC-2 back-propagation processing**, after the observed signal has been corrected and interpolated onto a uniform BP-plane coordinate grid by `StationaryCorrector`.

This module implements the paper’s plane-wave expansion solution of the wave equation in vacuum:

\[
u(x,z)
=
\sum_m c_m
\exp
\left[
i k_{zm} z
+
i \sqrt{k^2-k_{zm}^2}(x-x_0)
\right]
\]

and its 3D counterpart:

\[
u(x,y,z)
=
\sum_{mn} c_{mn}
\exp
\left[
i k_{zm}z
+
i k_{yn}y
+
i \sqrt{k^2-k_{zm}^2-k_{yn}^2}(x-x_0)
\right]
\]

The paper states that this approach neglects back-scattered waves and is equivalent to solving the parabolic wave equation.

The public interface must follow the design exactly:

```text
class FftPropagator {
    +__init__(wavelength_m: float)
    +propagate_2d(field_z: ndarray, z_m: ndarray, distance_m: float) ndarray
    +backpropagate_2d(field_z: ndarray, z_m: ndarray, distance_m: float) ndarray
    +propagate_3d(field_yz: ndarray, y_m: ndarray, z_m: ndarray, distance_m: float) ndarray
    +amplitude_variance(field: ndarray) float
}
```

No additional public methods should be required by downstream modules.

---

## 2. Paper Methodology Alignment

### 2.1 Propagation Representation

The paper uses complex radio-wave amplitude:

\[
u = A \exp(i\phi)
\]

where:

- \(A\) is amplitude,
- \(\phi\) is phase in radians,
- \(k = 2\pi / \lambda\),
- \(\lambda\) is the GNSS carrier wavelength.

This module should operate directly on complex arrays already constructed by upstream modules.

For synthetic experiments, `PhaseScreenGenerator` will produce phase-screen outputs such as:

\[
u_{\mathrm{out}}(z) = A_0 \exp[i k S_0(z)]
\]

or equivalently:

\[
u_{\mathrm{out}}(z) = A_0 \exp[i\phi(z)]
\]

For real data, `StationaryCorrector.correct(...)` will produce a `CorrectedSignal` containing:

- `z_m`,
- `complex_signal`,
- corrected phase,
- uncorrected amplitude,
- curvature-corrected signal.

`FftPropagator` should not perform stationary-transmitter correction, curvature correction, amplitude normalization, or detrending. Those responsibilities belong to other modules in the design.

## 5. `propagate_2d(field_z, z_m, distance_m)`

### 5.1 Intended Use

This method propagates a 1D complex field along the propagation direction by distance `distance_m`.

It implements:

\[
u(x,z)
=
\mathcal{F}^{-1}
\left[
\mathcal{F}\{u(z)\}
\exp
\left(
i k_x \Delta x
\right)
\right]
\]

where:

\[
k_x = \sqrt{k^2-k_z^2}
\]

and:

\[
\Delta x = \text{distance\_m}
\]

For forward propagation from phase screen to receiver:

\[
\Delta x > 0
\]

For back propagation, the design provides a separate method `backpropagate_2d`.

---

### 5.2 Input Requirements

Inputs:

```text
field_z: ndarray
z_m: ndarray
distance_m: float
```

Expected:

- `field_z` is a 1D complex NumPy array.
- `z_m` is a 1D real NumPy array.
- `len(field_z) == len(z_m)`.
- `z_m` is uniformly spaced.
- Units of `z_m` are meters.
- Units of `distance_m` are meters.

The FFT plane-wave method assumes uniform spatial sampling. If `z_m` is not uniform, propagation will be physically invalid. Since real-data interpolation to a uniform grid is handled by `StationaryCorrector`, this module should verify or at least defensively check approximate uniformity.

Recommended logic:

1. Convert inputs to NumPy arrays.
2. Verify dimensionality.
3. Verify nonzero length.
4. Verify matching sizes.
5. Compute:

\[
\Delta z = \mathrm{median}(\mathrm{diff}(z_m))
\]

6. Check that all spacings are close to `Δz`.

If nonuniform, raise a clear error rather than silently producing incorrect propagation.

---

### 5.3 Spatial Frequency Grid

Use FFT frequencies:

\[
f_z = \mathrm{fftfreq}(N, d=\Delta z)
\]

where \(f_z\) is in cycles/m.

Convert to angular spatial wavenumber:

\[
k_z = 2\pi f_z
\]

The propagation factor is:

\[
H(k_z, \Delta x)
=
\exp
\left[
i \sqrt{k^2-k_z^2}\Delta x
\right]
\]

Then:

\[
u_{\mathrm{prop}}(z)
=
\mathrm{IFFT}
\left[
\mathrm{FFT}(u(z)) H(k_z,\Delta x)
\right]
\]

---

### 5.4 Evanescent Components

For frequencies where:

\[
|k_z| > k
\]

the square root becomes imaginary. These are evanescent or non-propagating components.

The task explicitly requires:

> Handle evanescent/invalid frequencies robustly by zeroing, clipping, or using complex square root consistently.

For reproduction fidelity and numerical stability, the preferred behavior should be documented clearly.

Recommended implementation logic:

- Compute a Boolean mask:

\[
\mathrm{propagating} = k_z^2 \leq k^2
\]

- For propagating modes:

\[
k_x = \sqrt{k^2-k_z^2}
\]

- For non-propagating modes, set the transfer function to zero:

\[
H = 0
\]

Rationale:

1. The paper’s method focuses on propagating plane-wave components and neglects backscatter.
2. GNSS wavelength is small compared to scintillation scales, so physically meaningful phase-screen spectra should mainly occupy propagating modes.
3. Zeroing avoids exponential blow-up during negative-distance back propagation.

Alternative complex-square-root handling can be useful in special diffraction simulations, but for the main reproduction pipeline zeroing evanescent modes is safer.

This behavior should be documented in the module docstring.

---

### 5.5 Sign Convention

For forward propagation:

```text
propagate_2d(..., distance_m > 0)
```

uses:

\[
H = \exp(i k_x \Delta x)
\]

For negative `distance_m`, the same method mathematically performs backward propagation:

\[
H = \exp(-i k_x |\Delta x|)
\]

However, the design includes a dedicated method:

```text
backpropagate_2d(field_z, z_m, distance_m)
```

Therefore downstream code should normally call `backpropagate_2d` for BP.

---

## 6. `backpropagate_2d(field_z, z_m, distance_m)`

### 6.1 Intended Use

This method is used in:

- synthetic BP experiments,
- real-data BP inner loop inside `BackpropProcessor`.

It should propagate the receiver-observed complex field backward toward the transmitter / phase screen.

The paper states:

> The same solution is applied for BP, by using \(u(z)\) on the BP trajectory \(x=x_{BP}\) as the boundary condition and replacing \(x-x_0\) by \(x-x_{BP}\).

If the caller passes a positive screen distance from receiver toward transmitter, then the method should internally use:

\[
-\text{distance\_m}
\]

Thus:

```text
backpropagate_2d(field_z, z_m, distance_m)
```

should be equivalent to:

```text
propagate_2d(field_z, z_m, -distance_m)
```

where `distance_m` is positive.

---

### 6.2 Expected Behavior

For a synthetic single phase-screen experiment:

1. Generate phase screen.
2. Forward propagate from screen to receiver by \(L\).
3. Backpropagate receiver field over candidate distances.
4. Compute amplitude variance:

\[
V(L) = \frac{\langle A^2\rangle}{\langle A\rangle^2} - 1
\]

Expected:

- \(V(L)\) should reach a minimum near the true phase-screen distance.
- This reproduces the behavior shown in paper Figure 4.

---

### 6.3 Distance Validation

The method may accept any real `distance_m`, but normal usage is:

\[
distance\_m > 0
\]

If `distance_m == 0`, output should equal input up to numerical FFT roundoff.

---

## 7. `propagate_3d(field_yz, y_m, z_m, distance_m)`

### 7.1 Intended Use

This method supports the paper’s 3D forward propagation synthetic experiment for two phase screens with different irregularity orientations.

The paper uses 3D propagation for Section 2.3:

\[
u(x,y,z)
=
\sum_{mn} c_{mn}
\exp
\left[
i k_{zm}z
+
i k_{yn}y
+
i \sqrt{k^2-k_{zm}^2-k_{yn}^2}
(x-x_0)
\right]
\]

This is required for synthetic validation of the finding that, for two regions with equal \(\sigma_\phi\) but different scan angles, only the phase screen with smaller \(\alpha\) is correctly localized.

The relevant configuration is:

```yaml
synthetic_experiments:
  modeled_cases:
    two_phase_screens_different_orientation:
      enabled: true
      forward_propagation_dimension: "3D"
      back_propagation_dimension: "2D"
      alpha_cases_deg:
        - [30, 60]
        - [60, 30]
```

---

### 7.2 Input Requirements

Inputs:

```text
field_yz: ndarray
y_m: ndarray
z_m: ndarray
distance_m: float
```

Expected:

- `field_yz` is a 2D complex array.
- One axis corresponds to `y_m`.
- One axis corresponds to `z_m`.

The implementation must consistently define the axis convention. Recommended:

```text
field_yz.shape == (len(y_m), len(z_m))
```

Then:

- axis 0 is y,
- axis 1 is z.

This convention should be documented clearly because downstream synthetic experiments must construct fields accordingly.

Inputs must be uniformly spaced in both `y_m` and `z_m`.

---

### 7.3 3D Frequency Grid

Compute:

\[
k_y = 2\pi \mathrm{fftfreq}(N_y, d=\Delta y)
\]

\[
k_z = 2\pi \mathrm{fftfreq}(N_z, d=\Delta z)
\]

Create 2D meshgrid:

\[
K_y, K_z
\]

Then:

\[
k_x = \sqrt{k^2-K_y^2-K_z^2}
\]

The transfer function is:

\[
H(K_y,K_z,\Delta x)
=
\exp(i k_x \Delta x)
\]

For non-propagating modes:

\[
K_y^2 + K_z^2 > k^2
\]

set:

\[
H = 0
\]

Then:

\[
u_{\mathrm{prop}}(y,z)
=
\mathrm{IFFT2}
\left[
\mathrm{FFT2}(u(y,z))H(K_y,K_z)
\right]
\]

---

### 7.4 Back Propagation in 3D

The design does not expose a public `backpropagate_3d` method. The paper’s misaligned-screen experiment uses:

- 3D forward propagation,
- extraction of a 1D signal along the receiver trajectory,
- projection to a 2D BP plane,
- 2D back propagation.

Therefore `propagate_3d` is sufficient.

---

## 8. `amplitude_variance(field)`

### 8.1 Paper Definition

The paper defines normalized BP amplitude variance:

\[
V =
\frac{\langle A^2\rangle}{\langle A\rangle^2}
-
1
\]

where:

\[
A = |u|
\]

This quantity is central to geolocation. The phase-screen location is inferred from the minimum of \(V(L)\).

This method must implement exactly:

```text
A = abs(field)
V = mean(A**2) / mean(A)**2 - 1
```

not the conventional variance normalized by mean intensity.

---

### 8.2 Input Shape

`field` may be:

- 1D complex array from `backpropagate_2d`,
- 2D complex array from `propagate_3d`.

For the main BP geolocation, it will generally be 1D.

For multidimensional input, the method can compute the variance over all elements unless the caller slices the field beforehand.

---

### 8.3 Numerical Safety

If:

\[
\langle A\rangle = 0
\]

then \(V\) is undefined.

Recommended behavior:

- Return `np.nan`, or
- raise a clear error.

For pipeline robustness, returning `np.nan` is likely preferable, because `QualityController.find_global_local_minimum(...)` can reject invalid curves. However, this behavior should be documented.

Also handle non-finite values:

- If `field` contains NaN or Inf, the output should become NaN unless explicitly filtered.
- Do not silently ignore NaNs unless a policy is defined elsewhere.

---

## 9. Interaction With `BackpropProcessor`

`BackpropProcessor.compute_v_curve(...)` will use this module in the inner BP loop.

Expected call sequence:

1. `StationaryCorrector.correct(...)` returns `CorrectedSignal`.
2. `CorrectedSignal.z_m` is a uniform 1D spatial grid.
3. `CorrectedSignal.complex_signal` is the corrected complex field.
4. `BackpropProcessor` calls:

```text
bp_field = propagator.backpropagate_2d(
    corrected.complex_signal,
    corrected.z_m,
    screen_distance_m
)
```

5. Then:

```text
v_value = propagator.amplitude_variance(bp_field)
```

6. The resulting \(V(L)\) curve is sent to `QualityController`.

`FftPropagator` should not know about:

- \(L_{\mathrm{mf}}\),
- \(D(L_{\mathrm{mf}})\),
- \(Q\),
- magnetic-field direction,
- scan angle \(\alpha\),
- tangent-point height,
- local time,
- multi-valued geolocations.

Those are handled by other modules in the design.

## 11. FFT Normalization and Numerical Consistency

NumPy/SciPy FFT conventions are acceptable if used consistently:

\[
u_{\mathrm{prop}} =
\mathrm{ifft}
\left[
\mathrm{fft}(u)H
\right]
\]

Because forward and backward operations use the same convention, no additional normalization factor is required.

Important consistency requirements:

1. Use the same spatial grid before and after propagation.
2. Do not reorder FFT frequencies manually unless also reordering the spectrum.
3. Use `fftfreq`, not `fftshift`, unless carefully applying shifts throughout.
4. Return an array with the same shape as the input field.
5. Preserve complex dtype.

---

## 12. Sampling and Aliasing Considerations

The paper’s synthetic thermal-noise modeling used:

```yaml
synthetic_experiments:
  modeled_cases:
    thermal_noise:
      modeled_spatial_interval_km: 40
      windowed_spatial_interval_km: 35
      internal_grid_step_m: 10
```

This means synthetic grids may be finely sampled.

However, real COSMIC-2 10-second windows have sample spacings derived from projected scan velocity and sampling rate. `StationaryCorrector` is responsible for interpolation to a uniform grid.

`FftPropagator` should not resample internally.

Potential aliasing issues:

- If spatial sampling is too coarse, high spatial frequencies may alias.
- If the spatial window is too short, \(V(L)\) may suffer edge artifacts.
- The paper mitigates edge effects in synthetic noise modeling by windowing the spatial interval.

This module should remain a pure propagator and not apply windowing unless explicitly added elsewhere.

## 13. Edge Cases and Failure Modes

### 13.1 Empty Input

If `field_z` or `field_yz` is empty, propagation is invalid.

Recommended behavior:

- Raise `ValueError`.

### 13.2 Nonuniform Grid

The FFT method requires uniform spacing.

Recommended behavior:

- Raise `ValueError` with a message indicating that `z_m`, `y_m`, or both must be uniformly spaced.

### 13.3 Too Few Samples

For meaningful FFT propagation, at least several samples are needed.

Recommended behavior:

- Raise `ValueError` for arrays with fewer than two samples along a required axis.

### 13.4 Zero or Negative Wavelength

Invalid physical input.

Recommended behavior in `__init__`:

- Raise `ValueError` if `wavelength_m <= 0`.

### 13.5 Non-Finite Values

If input fields or grids contain NaN/Inf:

- Raise `ValueError` for grids.
- For fields, either raise or propagate NaNs.

For reproducible processing, raising early is preferable for propagation methods, while `amplitude_variance` can return NaN for invalid fields if desired.

## 15. Important Non-Responsibilities

This module must not implement logic belonging to other modules.

It should not:

1. Generate phase screens.  
   That belongs to `PhaseScreenGenerator`.

2. Compute \(\sigma_\phi\) or \(S_4\).  
   That belongs to `ScintillationCalculator`.

3. Smooth \(V(L)\).  
   That belongs to `QualityController.smooth_v_curve(...)`.

4. Find local minima or \(Q\).  
   That belongs to `QualityController`.

5. Define BP plane geometry.  
   That belongs to `BpPlaneBuilder`.

6. Apply stationary-transmitter correction.  
   That belongs to `StationaryCorrector`.

7. Apply curvature correction.  
   That belongs to `StationaryCorrector.apply_curvature_correction(...)`.

8. Determine geolocation or zero crossings.  
   That belongs to `BackpropProcessor`, `QualityController`, and `Geolocator`.

Keeping this separation is essential to follow the provided design.

## 16. Recommended Implementation Strategy

Internally, the implementation can be organized with private helper logic, but downstream code should only rely on the public interface from the design.

Recommended internal flow for `propagate_2d`:

1. Validate wavelength was initialized.
2. Convert `field_z` and `z_m` to arrays.
3. Validate shapes and uniform spacing.
4. Compute `dz`.
5. Compute FFT frequency grid.
6. Convert to angular wavenumber \(k_z\).
7. Build propagation transfer function.
8. FFT field.
9. Multiply spectrum by transfer function.
10. Inverse FFT.
11. Return propagated complex field.

Recommended internal flow for `backpropagate_2d`:

1. Call `propagate_2d(field_z, z_m, -distance_m)`.

Recommended internal flow for `propagate_3d`:

1. Validate 2D input.
2. Validate `y_m` and `z_m`.
3. Compute `dy`, `dz`.
4. Compute \(k_y\), \(k_z\) grids.
5. Build 2D transfer function.
6. Apply `fft2`, multiply, then `ifft2`.
7. Return propagated 2D complex field.

Recommended internal flow for `amplitude_variance`:

1. Compute amplitude:

\[
A = |u|
\]

2. Compute:

\[
V = \frac{\mathrm{mean}(A^2)}{\mathrm{mean}(A)^2} - 1
\]

3. Return scalar float.

## 17. Reproduction-Critical Details

The following choices are especially important for reproducing the paper:

1. **Use amplitude, not intensity, in the denominator of \(V\)**:

\[
V = \frac{\langle A^2\rangle}{\langle A\rangle^2}-1
\]

2. **Back propagation must use the opposite sign of propagation distance.**

3. **Spatial frequencies must be angular wavenumbers**, not cycles per meter:

\[
k_z = 2\pi f_z
\]

4. **The transfer phase must use**:

\[
\sqrt{k^2-k_z^2}
\]

or in 3D:

\[
\sqrt{k^2-k_y^2-k_z^2}
\]

5. **The method assumes uniform spatial sampling.**

6. **Evanescent modes must be handled consistently**, preferably by zeroing them for numerical stability.

7. **No real-data geometry correction should be embedded here**, because the paper separates propagation from observational-geometry correction.

---

## 18. Expected Role in End-to-End Reproduction

When implemented correctly, this module enables the rest of the pipeline to reproduce the paper’s central BP result:

- after forward propagation through a thin phase screen, amplitude fluctuations grow with distance;
- after back propagation from receiver toward transmitter, amplitude variance decreases toward the phase screen;
- the minimum of:

\[
V(L)
=
\frac{\langle A^2\rangle}{\langle A\rangle^2}-1
\]

identifies the apparent irregularity distance \(L_0\);
- the real geolocation algorithm then finds where:

\[
D(L_{\mathrm{mf}})
=
L_0 - L_{\mathrm{mf}}
=
0
\]

This module is therefore computationally central, but scientifically narrow: it should implement only the vacuum wave propagation operator and the normalized amplitude variance used by the paper.
