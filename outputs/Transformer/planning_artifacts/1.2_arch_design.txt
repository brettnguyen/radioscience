[Implementation approach]
We will implement a modular Transformer system using PyTorch and its ecosystem. The solution leverages open-source libraries such as PyTorch for model building and training, Hugging Face’s tokenizers for BPE/word-piece tokenization, and SacreBLEU for evaluation. The overall architecture is split into distinct components: data loading (for both translation and parsing tasks), the Transformer model (with encoder, decoder, multi-head attention, feed-forward, and positional encoding modules), a trainer that manages the training loop (including the learning rate scheduler, optimizer, dropout, and checkpointing), and an evaluation module that performs inference (using beam search) and computes BLEU and F1 scores. This simple, modular design allows for easy configuration adjustments and integration of additional experiments or ablation studies.

%% Mermaid Sequence Diagram
sequenceDiagram
    participant M as Main
    participant DL as DatasetLoader
    participant TM as TransformerModel
    participant TR as Trainer
    participant EV as Evaluation
    participant CM as CheckpointManager

    M->>DL: load_translation_data() or load_parsing_data()
    DL-->>M: Return train and test DataLoaders
    M->>TM: Initialize TransformerModel(config.params)
    M->>TR: Initialize Trainer(TM, {train_data, val_data}, config)
    loop Training Loop
        TR->>TM: forward(src, tgt)
        TM-->>TR: predictions
        TR->>TM: Backpropagation and optimizer step
        TR->>CM: Check within each epoch: save_checkpoint(model, epoch)
    end
    TR-->>M: Training complete
    M->>EV: evaluate_translation(test_data)
    EV->>TM: forward(test_data.input)
    TM-->>EV: Return predictions
    EV-->>M: Return BLEU scores
    M->>EV: evaluate_parsing(test_data)
    EV->>TM: forward(test_data.input)
    TM-->>EV: Return predictions
    EV-->>M: Return Parsing F1 scores
