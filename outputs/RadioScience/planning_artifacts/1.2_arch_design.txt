----------------------------------------
[Implementation approach]
Build a simple Python command-line research pipeline with two modes: (1) synthetic validation experiments for the back-propagation method, and (2) COSMIC-2 high-rate data processing for real geolocation products. Use NumPy/SciPy for signal processing and propagation, xarray/netCDF4/h5py for COSMIC-2 files, pandas/pyarrow for tabular outputs, astropy or sgp4/gnss utilities as needed for orbit interpolation, pymap3d/pyproj for geodetic transforms, ppigrf or pyIGRF for IGRF-13 magnetic-field direction, matplotlib/cartopy for maps, and pydantic/yaml for configuration. The architecture is intentionally modular but small: data loading, geometry, signal preprocessing, propagation, geolocation, quality control, synthetic experiments, and plotting. The first implementation should use FFT/plane-wave back propagation with the paper's Option 2 stationary-transmitter correction, while keeping a placeholder interface for Kirchhoff integral validation later.

    else Real COSMIC-2 processing mode
        MAIN->>REAL: process_year(year, signal)
        REAL->>CL: iter_windows(year, signal)
        loop For each COSMIC-2 event file
            CL->>CL: load_event(path)
            CL->>OL: interpolate_rx_states(window)
            OL-->>CL: rx_states
            CL->>OL: interpolate_tx_states(window)
            OL-->>CL: tx_states
            CL->>CL: segment_to_windows(event_data, 10)
            CL-->>REAL: SignalWindow iterator
        end
        loop For each 10-second SignalWindow
            REAL->>GEO: process_window(window, wavelength_m)
            GEO->>SP: prepare_window(window)
            SP->>SP: detrend_phase(phase_rad, times)
            SP->>SP: normalize_amplitude(amplitude)
            SP-->>GEO: prepared_window
            GEO->>SC: compute(prepared_window)
            SC->>SC: compute_sigma_phi(phase_rad)
            SC->>SC: compute_s4(amplitude)
            SC-->>GEO: ScintillationMetrics
            GEO->>QC: passes_pre_bp(metrics, tangent_height_km)
            alt Pre-BP QC failed
                QC-->>GEO: false, rejection_reason
                GEO-->>REAL: rejected GeolocationResult
            else Pre-BP QC passed
                GEO->>BP: compute_d_curve(prepared_window, wavelength_m)
                loop l_mf_km = 100 to 6000 step 100
                    BP->>PB: build(window, l_mf_km)
                    PB->>LOS: candidate_position(window, l_mf_km)
                    LOS-->>PB: candidate_position_m
                    PB->>MF: field_unit(mid_time, candidate_position_m)
                    MF-->>PB: magnetic_unit
                    PB->>PB: compute_scan_angle(window, plane)
                    PB-->>BP: BpPlaneGeometry
                    BP->>BP: compute_v_curve(window, plane, wavelength_m)
                    loop screen_distance_km = 100 to 6000 step 100
                        BP->>ST: correct(window, plane, screen_distance_km, option=2)
                        ST->>ST: apply_tx_rx_correction(window, plane, screen_distance_km, option=2)
                        ST->>ST: apply_phase_correction(phase_rad, original_path_m, corrected_path_m, wavelength_m)
                        ST->>ST: apply_curvature_correction(phase_rad, z_m, alpha_rad, radius_m, wavelength_m)
                        ST-->>BP: CorrectedSignal
                        BP->>FP: backpropagate_2d(complex_signal, z_m, distance_m)
                        FP-->>BP: bp_field
                        BP->>FP: amplitude_variance(bp_field)
                        FP-->>BP: v_raw_value
                    end
                    BP->>QC: smooth_v_curve(distances_km, v_raw)
                    QC-->>BP: v_smooth
                    BP->>QC: find_global_local_minimum(distances_km, v_smooth)
                    QC-->>BP: l0_km, v0, valid_minimum
                    BP->>QC: compute_q(distances_km, v_smooth, l0_km)
                    QC-->>BP: l1_km, l2_km, v1, v2, q
                    BP-->>BP: store BpCurve and D = l0_km - l_mf_km
                end
                BP->>QC: find_zero_crossings(d_curve)
                QC-->>BP: zero_crossings_km
                BP-->>GEO: DCurve

                GEO->>LOS: geolocation_from_distance(window, distance_km)
                LOS->>CT: ecef_to_geodetic(position_m)
                CT-->>LOS: lat_deg, lon_deg, alt_km
                LOS-->>GEO: lat_deg, lon_deg, alt_km
                GEO->>QC: passes_final(result)
                QC-->>GEO: accepted_or_rejected
                GEO-->>REAL: GeolocationResult
            end
        end
        REAL-->>MAIN: results DataFrame
        MAIN->>WR: write_dataframe(results, geolocations.parquet)
        MAIN->>REAL: make_monthly_bins(results)
        REAL-->>MAIN: monthly_binned_counts

        MAIN->>PLT: plot_monthly_maps(monthly_binned_counts, output_dir)
    else L1/L2 comparison mode

[Anything UNCLEAR]
Several implementation details are ambiguous in the paper and should be verified against COSMIC-2/CDAAC documentation or tested by sensitivity analysis: the exact HR file schema and whether GLONASS phases are already demodulated/connected; the exact phase detrending method used before computing sigma_phi; whether final operational BP used FFT plane-wave propagation with stationary-correction Option 2 or Kirchhoff diffraction integral with Option 1; the exact normalization of SNR/amplitude before BP; the missing Kirchhoff integral equation in the parsed text; the exact local-time reference point for post-sunset filtering; the sign convention in the published geolocation vector formula; and the discrepancy between the abstract's '2-month' L1/L2 comparison and the body text's February 19 to March 1, 2021 period.
