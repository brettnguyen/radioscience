## Logic Analysis: `src/core/constants.py`

### Purpose

`src/core/constants.py` should provide project-wide physical constants, GNSS carrier definitions, unit-conversion helpers, and paper/configuration default method values used by multiple modules.

This module must be dependency-light and must not import any internal project modules, because it will be imported by propagation, signal processing, geometry, geolocation, quality control, and experiment modules. It should avoid circular dependencies.

The file should distinguish between:

1. **Physical constants**  
   Stable values such as speed of light, WGS84 ellipsoid constants, and GNSS carrier frequencies.

2. **Paper/config defaults**  
   Default values matching `config.yaml` and the paper, used primarily as fallback values for `AppConfig` validation or synthetic experiments.

3. **Runtime configuration values**  
   Operational processing should use `AppConfig` values loaded from `config.yaml`, not hard-coded constants scattered across the codebase.

---

## 1. Dependency Rules

`src/core/constants.py` should only depend on:

- Python standard library, preferably `math`.

It should not import:

- `numpy`
- `pandas`
- `pyproj`
- project modules such as `src.config`, `src.core.types`, etc.

Reason:

- This file is a foundational dependency.
- Importing project modules here would create circular dependency risk.
- NumPy is unnecessary for scalar constants.

---

## 2. Unit Conventions

All constants should use explicit unit suffixes in their names.

Recommended naming convention:

| Quantity type | Suffix |
|---|---|
| meters | `_M` |
| kilometers | `_KM` |
| seconds | `_S` |
| hertz | `_HZ` |
| megahertz | `_MHZ` if used |
| radians | `_RAD` |
| degrees | `_DEG` |
| dimensionless | no unit suffix or descriptive suffix |

This matches the project’s shared convention:

- ECEF positions are in meters.
- BP/geolocation distances are usually in kilometers in public APIs.
- Wavelengths are in meters.
- Phase is in radians.
- Frequencies are in hertz.

## 3. Physical Constants

### 3.1 Speed of Light

Required for GNSS wavelength calculation and propagation wave number:

\[
\lambda = \frac{c}{f}
\]

\[
k = \frac{2\pi}{\lambda}
\]

Use:

```text
SPEED_OF_LIGHT_MPS = 299792458.0
```

This is exact in SI units.

Used by:

- `src/core/constants.py` wavelength definitions
- `src/propagation/fft_propagator.py`
- `src/propagation/phase_screen.py`
- `src/geolocation/stationary_correction.py`

## 4. Earth Reference Ellipsoid Constants

Although `pyproj` or `pymap3d` should perform primary geodetic transformations, basic WGS84 constants are useful for fallback calculations, diagnostics, and tests.

Recommended constants:

```text
WGS84_A_M = 6378137.0
WGS84_F = 1 / 298.257223563
WGS84_B_M = WGS84_A_M * (1 - WGS84_F)
WGS84_E2 = WGS84_F * (2 - WGS84_F)
```

Purpose:

- Tangent-height sanity checks.
- Fallback geodetic conversions if external libraries are unavailable.
- Unit tests for coordinate transformations.

Important:

- Real coordinate conversion logic belongs in `src/geometry/coordinates.py`, not here.
- This file should only expose constants.

## 6. GNSS Carrier Frequencies and Wavelengths

The paper uses GNSS RO high-rate scintillation data from COSMIC-2, with GPS and GLONASS sampling rates specified in `config.yaml`:

```yaml
data:
  cosmic2:
    sampling_rates_hz:
      gps: 50
      glonass: 100
```

The synthetic experiments use a default signal:

```yaml
synthetic_experiments:
  wavelength:
    default_signal: "L1"
```

Therefore `constants.py` should expose GNSS frequency and wavelength definitions.

---

### 6.1 GPS Frequencies

GPS L1:

\[
f_{L1} = 1575.42 \text{ MHz}
\]

GPS L2 / L2C:

\[
f_{L2} = 1227.60 \text{ MHz}
\]

Recommended constants:

```text
GPS_L1_FREQUENCY_HZ = 1575.42e6
GPS_L2_FREQUENCY_HZ = 1227.60e6
GPS_L2C_FREQUENCY_HZ = GPS_L2_FREQUENCY_HZ
```

Corresponding wavelengths:

```text
GPS_L1_WAVELENGTH_M = SPEED_OF_LIGHT_MPS / GPS_L1_FREQUENCY_HZ
GPS_L2_WAVELENGTH_M = SPEED_OF_LIGHT_MPS / GPS_L2_FREQUENCY_HZ
GPS_L2C_WAVELENGTH_M = GPS_L2_WAVELENGTH_M
```

Approximate values:

| Signal | Frequency | Wavelength |
|---|---:|---:|
| GPS L1 | 1575.42 MHz | 0.1902936728 m |
| GPS L2/L2C | 1227.60 MHz | 0.2442102134 m |

Use in:

- `main.py` when choosing `wavelength_m`
- `src/experiments/synthetic_experiments.py`
- `src/geolocation/geolocator.py`
- `src/propagation/fft_propagator.py`

---

### 6.2 GLONASS Frequencies

GLONASS uses FDMA frequencies depending on channel number \(k\). Therefore, unlike GPS, a single fixed L1 or L2 wavelength is not always correct.

Carrier definitions:

\[
f_{L1,k} = 1602.0\text{ MHz} + k \cdot 0.5625\text{ MHz}
\]

\[
f_{L2,k} = 1246.0\text{ MHz} + k \cdot 0.4375\text{ MHz}
\]

Recommended constants:

```text
GLONASS_L1_BASE_FREQUENCY_HZ = 1602.0e6
GLONASS_L1_CHANNEL_SPACING_HZ = 0.5625e6

GLONASS_L2_BASE_FREQUENCY_HZ = 1246.0e6
GLONASS_L2_CHANNEL_SPACING_HZ = 0.4375e6
```

Recommended helper logic:

- A helper can compute GLONASS frequency from channel number.
- A helper can compute GLONASS wavelength from channel number.

Conceptually:

```text
glonass_l1_frequency_hz(k) = GLONASS_L1_BASE_FREQUENCY_HZ + k * GLONASS_L1_CHANNEL_SPACING_HZ
glonass_l2_frequency_hz(k) = GLONASS_L2_BASE_FREQUENCY_HZ + k * GLONASS_L2_CHANNEL_SPACING_HZ
```

Important:

- If COSMIC-2 metadata provides exact GLONASS frequency or channel number, use that.
- Do not silently use a fixed nominal GLONASS wavelength for scientific processing unless channel information is unavailable and the approximation is explicitly logged.
- The paper does not specify GLONASS channel handling, so this should remain transparent and configurable.

---

### 6.3 Signal Name Mapping

It is useful to define a dictionary mapping common signal names to wavelengths.

Recommended conceptual mapping:

```text
GNSS_WAVELENGTHS_M = {
    "GPS_L1": GPS_L1_WAVELENGTH_M,
    "L1": GPS_L1_WAVELENGTH_M,
    "GPS_L2": GPS_L2_WAVELENGTH_M,
    "GPS_L2C": GPS_L2C_WAVELENGTH_M,
    "L2": GPS_L2_WAVELENGTH_M,
    "L2C": GPS_L2C_WAVELENGTH_M,
}
```

Caution:

- `"L1"` defaults to GPS L1 because the synthetic configuration specifies default signal `"L1"` and the paper’s synthetic wavelength is not explicitly stated.
- For real GLONASS data, signal-specific wavelength should be resolved using metadata where possible.

### 8.6 Thermal Noise Defaults

From `config.yaml`:

```yaml
thermal_noise:
  phase_screen_distance_km: 2000
  sigma_phi_cases_rad:
    - 1.65
    - 0.25
  snr_1hz_cases_vv:
    - 200
    - 400
    - 600
  average_snr_1hz_vv: 400
  assumed_sampling_rate_hz: 100
  assumed_projected_scan_velocity_km_s: 4
  modeled_spatial_interval_km: 40
  windowed_spatial_interval_km: 35
  internal_grid_step_m: 10
```

Paper:

- Noise modeling uses a phase screen at 2000 km.
- Uses \(\sigma_\phi=1.65\) rad and \(\sigma_\phi=0.25\) rad.
- Typical 1-Hz SNR values are 200–600 V/V, average 400 V/V.
- For \(f_s = 100\) Hz, effective SNR is reduced by \(\sqrt{100}=10\).
- For 1-Hz SNR = 400 V/V, effective SNR = 40 and thermal phase noise is:

\[
\sigma_\phi = \frac{1}{40} = 0.025 \ \mathrm{rad}
\]

Recommended constants:

```text
DEFAULT_NOISE_SCREEN_DISTANCE_KM = 2000.0
DEFAULT_NOISE_SIGMA_PHI_CASES_RAD = (1.65, 0.25)
DEFAULT_NOISE_SNR_1HZ_CASES_VV = (200.0, 400.0, 600.0)
DEFAULT_AVERAGE_SNR_1HZ_VV = 400.0
DEFAULT_NOISE_SAMPLING_RATE_HZ = 100.0
DEFAULT_PROJECTED_SCAN_VELOCITY_KM_S = 4.0
DEFAULT_NOISE_MODELED_INTERVAL_KM = 40.0
DEFAULT_NOISE_WINDOWED_INTERVAL_KM = 35.0
DEFAULT_NOISE_INTERNAL_GRID_STEP_M = 10.0
```

These are experiment defaults, not real-data QC thresholds.

## 14. What Should Not Be in `constants.py`

Do not include:

1. Project dataclasses  
   These belong in `src/core/types.py`.

2. YAML loading logic  
   This belongs in `src/config.py`.

3. Coordinate-transform functions  
   These belong in `src/geometry/coordinates.py`.

4. IGRF model calls  
   These belong in `src/geometry/magnetic_field.py`.

5. Propagation algorithms  
   These belong in `src/propagation/fft_propagator.py`.

6. Quality-control procedures  
   These belong in `src/geolocation/qc.py`.

7. Plotting constants tied to figure appearance  
   These belong in `src/visualization/plots.py`.

8. Runtime path values  
   Paths are configured in `config.yaml` and loaded by `AppConfig`.

## 17. Key Implementation Principle

`src/core/constants.py` should encode the paper/configuration defaults in one place, but scientific processing should still be driven by `AppConfig`.

In other words:

- `constants.py` provides stable defaults.
- `config.yaml` provides experiment/runtime values.
- `AppConfig` mediates between the two.
- Processing modules should avoid hard-coding thresholds directly.

This preserves reproducibility while allowing sensitivity tests for unclear paper details such as detrending method, BP propagation method, and amplitude normalization.
