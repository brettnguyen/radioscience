[Logic Analysis]
- ['config.py', 'Contains all configuration parameters such as hyperparameters, dataset paths, training settings, and model architecture details. This file is imported by all other modules to ensure consistency and ease of parameter tuning.']
- ['dataset_loader.py', "Defines the DatasetLoader class that handles loading and preprocessing for both translation and parsing tasks. Implements methods load_translation_data() and load_parsing_data() that utilize Hugging Face's tokenizers and PyTorch's DataLoader. Depends on torch, tokenizers, and configuration data from config.py."]
- ['model.py', 'Implements the TransformerModel class based on the paper. This class aggregates modules including encoder, decoder, multi-head attention, positional encoding, and feed-forward layers. Built using torch.nn.Module and organized into sub-components. Imports configuration details from config.py.']
- ['trainer.py', 'Contains the Trainer class that manages the training loop. This includes forward propagation, backpropagation, optimizer updates using Adam with a custom learning rate scheduler (linear warmup and inverse square-root decay), label smoothing, dropout, and checkpoint management. Imports the TransformerModel from model.py and data loaders from dataset_loader.py.']
- ['evaluation.py', 'Defines the Evaluation class responsible for running inference using beam search and assessing model performance. Computes BLEU scores for translation tasks (with sacrebleu) and F1 scores for constituency parsing. It imports the trained TransformerModel and evaluation configurations from config.py.']
- ['main.py', 'Serves as the entry point of the application. It loads the configuration, initializes the DatasetLoader to prepare data, instantiates the TransformerModel, triggers the training process via Trainer, and finally calls the Evaluation module for inference and metric computation. It orchestrates the overall workflow.']
